# BMO Firmware — Wiring & Secrets

This README is a quick reference for the BMO ESP32-C3 firmware: how to wire the
peripherals, and how to manage the gitignored `secrets.h` that holds the WiFi
credentials, dashboard URL, and dashboard-issued fingerprint.

For the full deployment story (dashboard side + firmware flashing), see
`docs/DEPLOY.md` in the dashboard repo. For end-to-end voice testing, see
`docs/HARDWARE-SMOKE-TEST.md` there.

---

## Hardware target

- **MCU**: ESP32-C3 Super Mini (Arduino framework, PlatformIO env
  `esp32c3_supermini`, see `platformio.ini`)
- **Display**: ST7735 160×128 RGB565 over hardware SPI @ 40 MHz
- **Audio out**: MAX98357A I²S amplifier + speaker
- **Audio in**: INMP441 I²S microphone
- **Touch**: TTP223 capacitive button on top of the BMO

## Pin map (matches `src/main.cpp`)

| Function           | GPIO  |
| ------------------ | ----- |
| Display CS         | GP7   |
| Display RESET      | GP10  |
| Display A0 / DC    | GP3   |
| Display SDA (MOSI) | GP6   |
| Display SCK        | GP4   |
| I²S BCLK (out)     | GP0   |
| I²S LRC (out)      | GP1   |
| I²S DOUT (out)     | GP2   |
| Touch button       | GP20  |

Power: VCC → 3V3, GND → GND, display LED → 3V3.

The mic uses the second I²S peripheral (pins assigned in the audio task; not
duplicated here to avoid drift — check `main.cpp` for the live values).

## Secrets workflow

The firmware needs four values that vary per device or per deploy:

- **`BMO_WIFI_SSID`** — the 2.4 GHz network the BMO joins
- **`BMO_WIFI_PASS`** — the password for that network
- **`BMO_DASHBOARD_URL`** — the deployed dashboard origin, e.g.
  `https://bmo-dashboard.vercel.app` (no trailing slash)
- **`BMO_FINGERPRINT`** — the plaintext fingerprint generated by the dashboard
  during onboarding (or rotation). The dashboard stores only the argon2id
  hash; the firmware sends the plaintext in the `X-BMO-Fingerprint` header.

These live in `include/secrets.h`, which is **gitignored** by
`firmware/bmo_face_anim/.gitignore`. The committed template is
`include/secrets.h.in`.

### One-time setup

```bash
cd firmware/bmo_face_anim
cp include/secrets.h.in include/secrets.h
$EDITOR include/secrets.h
```

Replace each `@PLACEHOLDER@` with a real value. Save.

### Build-time guard

`scripts/check_secrets_h.py` is the PlatformIO pre-build hook that catches a
missing or unfilled `include/secrets.h` before any C++ compilation runs. To
enable it, ensure `platformio.ini` includes:

```ini
extra_scripts = pre:scripts/check_secrets_h.py
```

With that line in place, before any compilation step the hook:

1. Fails the build if `include/secrets.h` is missing, with a multi-line error
   pointing back at this README and the template.
2. Fails the build if `include/secrets.h` exists but still contains
   `@PLACEHOLDER@` markers from the template.

You will see a clear PlatformIO error — not a confusing C++ compile error —
when these are misconfigured.

### Rotating the fingerprint

If you rotate the fingerprint in the dashboard's **Fingerprint** page, the old
value stops working within 5–60 seconds. The firmware will start getting `401`
responses and animate the `error` mood. Fix:

1. Copy the new plaintext from the rotate screen.
2. Paste into `include/secrets.h` (replace `BMO_FINGERPRINT`).
3. Re-flash:
   ```bash
   pio run -e esp32c3_supermini -t upload
   ```

### Never commit

`include/secrets.h` is in `.gitignore`. Sanity check before every commit:

```bash
git status -- include/secrets.h
# expect: nothing (file is ignored)

git ls-files include/secrets.h
# expect: empty output (file is not tracked)
```

If either of those returns the file, stop and fix the `.gitignore` before
pushing — the dashboard-issued fingerprint should never land in source control.

## Brain client

`src/bmo_brain_client.{h,cpp}` is the HTTPS client that POSTs captured audio to
`<BMO_DASHBOARD_URL>/api/brain` and streams the PCM16 reply into the I²S audio
output.

Public surface:

```cpp
namespace bmo {
  enum class BrainStatus { Idle, Listening, Thinking, Talking, Error };
  class BrainClient {
   public:
    BrainClient();
    void begin();                                    // WiFi + HTTPS init
    bool ask(const uint8_t* wavData, size_t wavLen); // POST + stream
    void onStatus(void (*cb)(BrainStatus));          // face hook
  };
}
```

The audio sink is a weak C symbol `bmo_audio_push_pcm16(const uint8_t*, size_t)`
that the audio task supplies. If the symbol isn't linked, chunks are silently
dropped — easy to spot during the smoke test (audio plays as nothing).

Timeouts (matching `docs/HARDWARE-SMOKE-TEST.md` expectations):

- TCP/TLS connect: 5 s
- First response byte: 10 s
- Total response stream: 30 s

TLS verification is currently `setInsecure()` — the fingerprint header is the
real authentication boundary. Cert pinning is a future task.
